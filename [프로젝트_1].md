# [프로젝트 1] 비교 분석 보고서

**'리뷰 이벤트 참여 고객 분류 및 알림 자동화'를 n8n과 Make에서 구현하여 비교 분석**

---

# 1. 구현한 자동화 개요

Google Form을 통해 접수된 리뷰 이벤트 참여 정보를 Google Sheets에서 감지한 후, 
리뷰 유형(포토 리뷰 / 텍스트 리뷰)에 따라 IF와 Router(Filter)를 이용하여 데이터를 분기하고, 
Gmail과 Discord 채널에 자동으로 메일/알림 메시지를 전송하는 자동화 워크플로우를 구현하였다.

---

# 2. 구현 결과 비교

| 비교 항목 | Make | n8n |
| --- | --- | --- |
| 구현 난이도 | 쉬움 | 보통 |
| UI 직관성 | 매우 직관적 | 다소 복잡 |
| 조건 분기 | Router와 Filter로 쉽게 구현 | IF/Switch 노드 활용 |
| Google Sheets 연동 | 매우 쉬움 | 쉬움 |
| 실행 채널 연동 | 간단 | 간단 |
| 오류 확인 | 실행 결과를 시각적으로 확인 가능 | 로그 확인 필요 |
| 초보자 적합성 | 높음 | 보통 |
| 확장성 | 보통 | 높음 |

---

# 3. Make의 장점

- 드래그 앤 드롭 방식으로 워크플로우를 쉽게 구성할 수 있었다.
- Router와 Filter를 이용한 조건 분기가 직관적이어서 초보자도 이해하기 쉬웠다.
- 실행 결과가 시각적으로 표시되어 어느 단계에서 성공하거나 실패했는지 빠르게 확인할 수 있었다.
- Google Sheets와 Discord 연결 과정이 비교적 간단하여 구현 시간이 짧았다.

### 단점

- 무료 플랜에서는 실행 횟수 제한이 있다.
- 복잡한 로직이나 세부적인 데이터 처리는 다소 제한적이다.
- 외부 API를 세밀하게 제어하는 기능은 n8n보다 많이 부족하다.

---

# 4. n8n의 장점

- 다양한 노드를 조합하여 복잡한 자동화를 자유롭게 설계할 수 있다.
- IF, Switch 등 조건 분기 기능이 다양하여 복잡한 업무 자동화에 적합하다.
- JavaScript(Code 노드)를 활용하여 데이터 가공과 로직 구현이 가능하다.
- 오픈소스 기반이므로 자체 서버(Self-hosting) 환경에서도 사용할 수 있어 확장성이 높다.

### 단점

- 처음 사용하는 경우 인터페이스가 다소 어렵게 느껴질 수 있다.
- 노드와 데이터 흐름을 이해하는 데 시간이 필요하다.
- 초보자는 오류 원인을 파악하는데 Make보다 시간이 더 걸릴 수 있다.

---

# 5. 프로젝트에 더 적합한 도구 분석

Google Form 응답을 조건에 따라 분류하고 Gmail과 Discord로 알림을 전송하는 비교적 단순한 자동화였다.

이러한 자동화에서는 **빠른 구현과 쉬운 유지보수**가 중요한 요소였으며, Make는 Router와 Filter를 이용해 조건 분기를 직관적으로 구성할 수 있어 구현이 수월했다. 또한 실행 결과를 시각적으로 확인할 수 있어 테스트와 오류 수정도 쉽게 진행할 수 있었다.

반면 n8n은 복잡한 데이터 처리나 다양한 시스템을 연결하는 대규모 자동화에는 강점이 있었지만, 
이번 프로젝트 수준에서는 그 장점을 충분히 활용하기 어려웠다.

---

# 6. 결론

이번 **'리뷰 이벤트 참여 고객 분류 및 알림 자동화'** 프로젝트에서는 **Make가 더 적합한 도구**라고 판단하였다.

그 이유는 아래와 같다.

- 직관적인 인터페이스로 빠르게 구현할 수 있었다.
- Router와 Filter를 이용한 조건 분기가 매우 쉬운 편이었다.
- Google Sheets와 Discord 연동이 간단했다.
- 실행 결과를 시각적으로 확인할 수 있어 테스트와 디버깅이 편리했다.

반면 n8n은 높은 확장성과 자유도를 제공하는 장점이 있어, 향후 여러 서비스와 연계하거나 확장된 업무 프로세스를 자동화하는 프로젝트에서는 더 적합한 선택이 될 수 있다.

---

# 회고

동일한 자동화를 두 도구에서 구현해 보면서, 자동화의 핵심은 **트리거(Trigger), 액션(Action), 조건 분기(Filter/IF)**를 적절히 설계하는 것이라는 점을 이해할 수 있었다. 또한 도구마다 인터페이스와 구현 방식은 달랐지만, 자동화의 기본 원리는 동일하다는 것을 체감할 수 있었다.

> 💡 **자동화의 기본 원리**
>
> **어떤 일이 발생하면(Trigger) → 정해진 규칙에 따라 판단하고(Filter/IF) → 원하는 작업(Action)을 자동으로 수행한다.**

이번 프로젝트에서는 **Make의 직관성과 쉬운 사용성**이 특히 인상적이었다. 비전공자인 나도 비교적 짧은 시간 안에 원하는 자동화를 구현할 수 있었고, Router와 Filter를 이용한 조건 분기가 이해하기 쉬웠다. 반면 n8n은 처음에는 인터페이스와 노드 개념이 다소 어렵게 느껴졌지만, 다양한 노드와 높은 확장성을 갖추고 있어 향후 더 복잡한 자동화를 설계할 때 활용 가치가 높은 도구라는 점을 확인할 수 있었다.
